## 一、游戏概述

### 1.1 项目愿景

一款融合**俯视角射击**、**饥荒式生存探索**、**Roguelike收集**的萌系卡通风格游戏。玩家在随机生成的世界中自由探索、收集资源、制造武器、击败敌人、建立聚落好感度，体验丰富的NPC互动与战斗乐趣。

**核心卖点**：

- 🎨 萌系卡通风格 + 整体换肤系统
- 🗺️ 饥荒式随机地图 + 种子复现
- 🔫 俯视角玩家射击 + 多种武器
- 👾 丰富NPC种类 + 独特掉落物
- 🏰 双游戏模式（自由探索 / 守护区域）

### 1.2 参考游戏

| 游戏       | 参考要素                                        |
| ---------- | ----------------------------------------------- |
| 逃离鸭科夫 | 俯视角射击、3D渲染、上帝视角                    |
| 小小部队   | 玩家控制移动、射击战斗                          |
| 饥荒       | 随机地图生成、种子系统、聚落/敌意机制、资源收集 |

### 1.3 目标用户

- 🎯 休闲射击游戏爱好者
- 🎯 生存建造游戏玩家
- 🎯 喜欢收集/养成系统的玩家
- 🎯 追求可爱画风的玩家

## 三、核心玩法设计

### 3.1 游戏模式（优先级确认）

> ⚠️ **开发优先级**：先实现 **A模式（自由探索模式）**，B模式（守护区域模式）后续扩展

#### A/B模式对比

| 维度         | A: 自由探索模式      | B: 守护区域模式     |
| ------------ | -------------------- | ------------------- |
| **核心体验** | 冒险探索、收集养成   | 纯粹战斗、波次挑战  |
| **地图**     | 随机生成大世界       | 固定或随机小战场    |
| **敌人**     | 聚落分布，自由AI     | 波次涌来，固定路线  |
| **玩家移动** | 全地图自由探索       | 限定区域内存动      |
| **目标**     | 探索、收集、挑战Boss | 坚守N波、生存下来   |
| **节奏**     | 慢热、策略性强       | 快节奏、爽感        |
| **类比**     | 饥荒式生存           | 塔防射击            |
| **时长**     | 长（可玩数小时）     | 短（每局15-30分钟） |

**简单理解**：

- **A模式** = 饥荒 + 射击RPG，到处跑、采集资源、打聚落、挑战Boss
- **B模式** = 纯射击塔防，站在圈里，敌人一波波来，守住就赢

#### 模式A：自由探索模式（Survival Mode）

```
游戏流程：
┌──────────────────────────────────────────────────────────┐
│  开始游戏 → 输入种子(可选) → 生成世界 → 自由探索        │
│       ↓                                              ↓   │
│  收集资源 ←─────────────────────────  击败敌人/完成任务   │
│       ↓                                              ↓   │
│  制造装备 ←─ 合成系统 ←─ 消耗材料 ←──────  获得掉落物   │
│       ↓                                              ↓   │
│  探索新区域 ←─ 解锁地图 ←────────────────  击败区域Boss  │
└──────────────────────────────────────────────────────────┘
```

**特点**：

- 玩家自由移动
- **视野系统**：默认有限视野（玩家周围可见），可配置开启全局可见（类似逃离鸭科夫）
- 敌人按聚落分布，玩家接近产生敌意
- 多种NPC聚落，可交互/交易/任务
- 资源收集 + 制造系统
- 角色成长（轻量，不繁琐）

#### 模式B：守护区域模式（Defense Mode）

```
游戏流程：
┌──────────────────────────────────────────────────────────┐
│  选择地图 → 放置玩家 → 开始波次 → 战斗 ←─┐              │
│       ↓                              │              │
│  升级装备 ←─── 击杀奖励 ←─────────────┘              │
│       ↓                                              ↓   │
│  波次难度递增 ←─────── 下一波 ←────────  击败当前波次   │
│                                                              │
│  失败：HP归零 / 胜利：坚守N波                             │
└──────────────────────────────────────────────────────────┘
```

**特点**：

- 玩家只能在指定区域内移动
- 敌人从多方向涌来
- 波次难度递增
- 击杀获得金币/资源
- 无探索，纯战斗爽感

### 3.2 操作方式

**俯视角射击标准方案**：

```
         鼠标点击/按住射击
```

| 数字键1-9 | 快捷切换背包武器 |
| Tab | 打开背包/暂停 |
| ESC | 暂停菜单 |

## 四、武器系统设计

> **MVP原则**：精简初版实现，快速验证核心玩法，为后续扩展留足空间

### 4.1 设计原则

- ✅ 玩家开局有初始武器
- ✅ 武器可通过多种方式获取
- ✅ 初版精简，后续扩展
- ✅ 弹药系统简化

### 4.2 武器获取方式

| 来源            | 说明             |
| --------------- | ---------------- |
| 🔫 **初始武器** | 玩家开局自带     |
| 📦 **材料制造** | 收集资源合成     |
| 💀 **敌人掉落** | 击杀敌人概率获得 |
| 🏪 **NPC交易**  | 用金币/材料交换  |
| 🎁 **地图探索** | 某些区域可捡到   |

### 4.3 MVP武器清单

#### 远程武器

| 武器   | 获取方式              | 弹药   | 特点               |
| ------ | --------------------- | ------ | ------------------ |
| 手枪   | ✅ 初始 + 合成 + 掉落 | 轻弹药 | 经典开局，稳定可靠 |
| 霰弹枪 | 合成 + 掉落           | 重弹药 | 近距离AOE          |
| 步枪   | 掉落 + NPC交易        | 轻弹药 | 中距离精准         |

#### 近战武器

| 武器 | 获取方式    | 耐久   | 特点               |
| ---- | ----------- | ------ | ------------------ |
| 拳头 | ✅ 初始     | 无限   | 默认武器，随时可用 |
| 短刀 | 合成 + 掉落 | 可修复 | 快速连击           |

**初版共5种武器，后续可扩展为：狙击枪、冲锋枪、弓箭、长剑、斧头等**

### 4.4 弹药系统（简化）

> 简化设计：统一弹药池，减少玩家管理负担

```
弹药类型（MVP简化版）：
┌─────────────────────────────────────────┐
│  🔵 轻弹药    ← 手枪、步枪、冲锋枪       │
│  🟠 重弹药    ← 霰弹枪、狙击枪          │
│  ⚪ 弓箭      ← 弓箭（可回收）           │
└─────────────────────────────────────────┘

好处：
- 只需管理2-3种弹药
- 背包压力小
- 后续可拆分/合并类型
```

### 4.5 武器切换与持有

```
武器栏位设计：
┌─────────────────────────────────────────┐
│  快捷栏 [1][2][3][4][5]                 │
│  当前装备 [🔫 手枪] [⚔️ -]             │
│                                         │
│  背包：可存放多把武器                   │
│  - 单手武器：占用1格                    │
│  - 双手武器：占用2格                    │
└─────────────────────────────────────────┘

切换逻辑：
- 按Q/E：在当前装备栏切换
- 按数字键：切换到快捷栏对应位置
- 点击背包武器：装备到空位
```

### 4.6 武器属性（数据结构）

```typescript
interface WeaponDefinition {
  id: string
  name: string
  type: 'ranged' | 'melee'

  // 战斗属性
  damage: number
  attackSpeed: number // 攻击间隔(秒)
  range: number // 攻击范围

  // 弹药/耐久
  ammoType?: string // 弹药类型（远程）
  magazineSize?: number // 弹匣容量
  reloadTime?: number // 换弹时间
  maxDurability?: number // 最大耐久（近战）

  // 持有方式
  holdType: 'one_hand' | 'two_hands'

  // 其他
  projectileSpeed?: number // 弹速（远程）
  knockback?: number // 击退力
  aoeRadius?: number // AOE半径
}
```

```
│                                                        │
│  5. 聚落放置                                            │
│     └→ 基于噪声的聚落位置                               │
│        (确保间距合理、主题匹配)                         │
│                                                        │
│  6. 路径连接                                            │
│     └→ 生成自然的路径网络                               │
```

### 10.1 存档数据结构

```typescript
interface SaveData {
  version: string
  timestamp: number

  // 世界信息
  world: {
    seed: string
    size: WorldSize
    difficulty: Difficulty
  }

  // 玩家数据
  player: {
    position: { x: number; y: number }
    rotation: number
    stats: PlayerStats
    inventory: InventoryData
    equipment: EquipmentSlots
    progression: PlayerProgression
  }

  // 世界状态
  worldState: {
    exploredAreas: string[]
    clearedDungeons: string[]
    npcRelations: Record<string, number>
    spawnedEntities: EntitySnapshot[]
    resourceRespawns: Record<string, number>
  }

  // 游戏模式特定
  modeData: SurvivalData | DefenseData
}

type SurvivalData = {
  mode: 'survival'
  quests: Quest[]
  discoveredLocations: string[]
}

type DefenseData = {
  mode: 'defense'
  currentWave: number
  highestWave: number
  totalKills: number
}
```

### 10.2 存档存储

> **存储策略**：单机离线存储优先，未来可扩展云存储

| 存储方式     | 说明                                      | 优先级       |
| ------------ | ----------------------------------------- | ------------ |
| LocalStorage | Web端默认存储，轻量存档（玩家数据、设置） | **当前实现** |
| IndexedDB    | 大型存档（地图数据、资源分布）            | **当前实现** |
| 云存档       | 未来联机模式扩展，需后端支持              | 后续扩展     |

```
存储架构：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   单机离线存储                    未来云存储            │
│        ↓                              ↓                 │
│   LocalStorage ←─── 存档格式 ───→ 云端数据库           │
│        ↓                              ↓                 │
│   当前实现                        联机模式时扩展        │
│                                                         │
└─────────────────────────────────────────────────────────┘

存档分离策略：
- 玩家数据（装备、背包、进度）：轻量 → LocalStorage
- 地图/世界数据：大量数据 → IndexedDB
- 存档格式统一，便于未来迁移到云端
```
