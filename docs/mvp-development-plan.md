# 🎮 MVP开发计划

> **项目代号**: 可爱幸存者 (Cute Survivor)
> **版本**: v0.1 MVP
> **更新时间**: 2026-03-26

---

## 一、MVP定义

**目标**：快速实现一个可运行的俯视角射击游戏核心玩法，验证核心循环。

### 核心循环（必须跑通）

```
玩家移动 → 射击敌人 → 击杀掉落 → 收集资源 → 制造弹药 → 继续战斗
```

## 三、技术架构

### 3.1 目录结构

```
src/
├── core/                    # 核心逻辑（平台无关）
│   ├── Game.ts            # 游戏主循环
│   ├── Entity.ts          # 实体基类
│   ├── Player.ts          # 玩家
│   ├── Enemy.ts           # 敌人
│   ├── Weapon.ts          # 武器
│   ├── Projectile.ts      # 弹道
│   ├── Inventory.ts       # 背包
│   ├── World.ts           # 世界
│   ├── MapGenerator.ts    # 地图生成
│   └── SaveSystem.ts      # 存档
│
├── renderer/              # 渲染层
│   ├── IRenderer.ts       # 渲染接口
│   └── ThreeJSRenderer.ts # Three.js实现
│
├── platforms/web/         # Web适配
│   ├── InputManager.ts    # 输入
│   └── Storage.ts         # 存储
│
└── main.ts               # 入口
```

### 3.2 核心类关系

```
Game (主循环)
├── Renderer (渲染)
├── InputManager (输入)
├── Player (玩家)
│   ├── Weapon (武器)
│   ├── Inventory (背包)
│   └── Stats (属性)
├── World (世界)
│   ├── MapGenerator (地图生成)
│   ├── Terrain[] (地形)
│   └── EntityManager (实体管理)
│       ├── Enemy[] (敌人)
│       └── Item[] (物品)
├── CombatSystem (战斗)
├── DropSystem (掉落)
├── CraftingSystem (合成)
└── SaveSystem (存档)
```

## 四、开发阶段

### 阶段一：核心骨架（预计1-2周）

**目标**：可移动、可射击、可击杀

```
Day 1-3: 项目搭建
├── 初始化Vite + TypeScript
├── 搭建目录结构
├── 基础渲染（Three.js俯视角）

Day 4-7: 玩家与战斗
├── 玩家移动（WASD）

├── 玩家射击（鼠标点击）
```

## 六、风险与对策

| 风险             | 影响 | 对策                       |
| ---------------- | ---- | -------------------------- |
| Three.js性能问题 | 中   | 先用简单模型，后续优化     |
| AI行为复杂       | 中   | MVP只做简单状态机          |
| 地图生成不稳定   | 低   | 先做固定地图，后续再加随机 |
| 存档数据丢失     | 中   | 频繁保存，多重备份         |

---

## 七、开发原则

1. **先跑通，再优化**：不追求完美，先实现功能
2. **数据驱动**：配置化设计，便于调整
3. **接口抽象**：核心逻辑与渲染分离
4. **快速迭代**：每周一个可运行版本
5. **持续测试**：每个功能立即测试
