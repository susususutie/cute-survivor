# Draft: Cute Survivor 开发执行计划

## 文档审阅结论

### 已确认决策
1. **阶段一地图**: 简化随机地图（只有草地+障碍物，不用固定地图）
2. **架构策略**: 先扁平后拆分 — MVP 直接实现，核心玩法验证后再重构为三层架构
3. **死亡流程**: Game Over 重开 — 死亡后回到主菜单，可选择继续（读档）或新游戏
4. **采集交互**: 靠近自动采集 — 走进资源点范围自动拾取，无需按键
5. **设计原则**: 实现简单、快速跑通、便于扩展、遵循业界通用做法

### 文档矛盾已解决
- 目录名: 统一使用 `docs/system/`（实际目录）
- 地图优先级: 阶段一做简化随机地图
- 武器数量: MVP 先做手枪+霰弹枪，步枪/短刀后续
- 存档策略: MVP 只用 LocalStorage，IndexedDB 后续
- 敌人数量: 明确为 4 种（每聚落 2 种）

### MVP 范围确认
- **P0**: 俯视角渲染、玩家移动、玩家射击、简单敌人AI、敌人掉落、物品拾取、弹药系统、存档
- **P1**: 随机地图、背包界面、合成系统、暂停菜单、更多敌人(4种)、霰弹枪、玩家HP
- **P2**: 完整武器(5种)、换肤系统、迷你地图、设置界面、守护区域模式

### 技术栈
- TypeScript + Three.js + Vite
- 扁平目录结构（MVP）
- LocalStorage 存档

### 待开发时确认（低风险）
- 玩家初始属性: HP=100, 速度=5, 基础伤害=10
- 碰撞体: 圆形碰撞，简单距离检测
- 伤害公式: damage = weaponDamage * (1 + proficiencyBonus)
- 游戏循环: requestAnimationFrame + delta time
